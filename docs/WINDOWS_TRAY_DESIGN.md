# PortClear Windows 托盘极简面板设计方案

## 结论

可以做，而且很适合 PortClear 的使用场景。

当前项目已经具备端口查询、占用分类、进程终止、管理员/非管理员检测等核心能力。Windows 托盘功能可以作为新的 Windows 适配层，在不改动核心逻辑的前提下，为用户提供一个常驻后台、右键唤出、快速查询和终止进程的极简入口。

推荐实现方式是：

- 使用 `pystray` 创建 Windows 任务栏托盘图标和右键菜单。
- 使用现有 `tkinter` 或 `customtkinter` 创建一个小型弹出面板。
- 查询和终止继续复用 `core.inspector.inspect_ports()` 与 `core.actions.kill_record()`。
- 查询操作继续放到后台线程，避免面板卡死。

## 目标体验

用户启动 PortClear 后，程序常驻 Windows 任务栏托盘区。用户右键点击托盘图标后，可以看到一个极简菜单：

- `快速查询端口`
- `打开主窗口`
- `以管理员身份运行`
- `退出`

点击 `快速查询端口` 后，弹出一个小面板：

- 输入端口号
- 点击 `查询`
- 显示占用该端口的进程列表
- 选中某个可终止进程
- 点击 `终止进程`
- 显示操作结果

如果查询到 `PID 4/System`，面板不提供终止按钮，而是显示 `HTTP.sys 诊断` 或提示用户使用管理员权限进一步排查。

如果查询到 `TIME_WAIT`，面板不提供“强制清理”，只显示等待或调优建议。

如果查询到 `CLOSE_WAIT`，面板提示这是应用未关闭连接导致的状态，优先终止对应进程。

## 可行性分析

### 技术可行

当前项目的核心逻辑已经和 GUI 解耦：

- `core.inspector.inspect_ports()` 可以按端口查询占用。
- `core.actions.kill_record()` 可以终止选中的进程。
- `core.privilege.get_privilege_info()` 可以判断管理员或普通用户。
- Windows GUI 已经使用后台线程查询，托盘极简面板可以复用同样模式。

托盘层只需要处理：

- 托盘图标创建
- 右键菜单
- 弹出小面板
- 面板和核心逻辑之间的调用

### 用户体验可行

端口清理工具本身是一个“用的时候很短、但可能经常需要”的工具。相比每次打开完整主窗口，托盘极简面板更适合开发场景：

- 启动后常驻后台
- 快速输入端口
- 快速查占用
- 快速杀进程
- 主窗口仅用于高级诊断

### 权限模型可行

托盘程序本身不能绕过 Windows 权限限制。非管理员启动时：

- 可以查询基础端口信息。
- 不能保证终止所有进程。
- 不能执行需要管理员权限的深度诊断。
- 不能修改系统参数。

因此托盘菜单建议提供 `以管理员身份运行` 项。点击后重新以管理员权限启动一个新实例，并提示用户当前实例可以退出。

## 推荐依赖

### 推荐方案：`pystray + pillow + tkinter`

依赖：

```bash
pip install pystray pillow
```

优点：

- 适合 Windows 托盘图标。
- API 简单。
- 可以和现有 `tkinter` 共存。
- 打包到 EXE 比较成熟。

缺点：

- 增加两个第三方依赖。
- 托盘事件循环和 Tk 主循环需要注意线程协调。

### 备选方案：`infi.systray`

优点：

- 使用简单。

缺点：

- 生态和维护活跃度不如 `pystray`。
- 后续扩展能力较弱。

### 不推荐：纯 Windows API 手写托盘

优点：

- 依赖最少。

缺点：

- 代码复杂。
- 易出兼容问题。
- 对当前项目收益不高。

## 建议文件结构

建议新增：

```text
windows/
  tray.py              # 托盘图标、右键菜单、生命周期管理
  quick_panel.py       # 极简查询面板
assets/
  portclear.ico        # 托盘图标
```

保留现有：

```text
windows/
  gui.py               # 完整主窗口
core/
  inspector.py         # 端口查询
  actions.py           # 终止进程和诊断建议
  privilege.py         # 权限检测
```

`main.py` 可以增加启动模式：

```bash
python main.py --tray
python main.py --gui
```

Windows 默认可以启动托盘模式，Linux 继续走 CLI。

## 交互流程

### 启动流程

1. 程序启动。
2. 检测当前是否 Windows。
3. 检测是否管理员。
4. 创建托盘图标。
5. 在托盘菜单中显示当前权限状态。
6. 程序进入托盘常驻状态。

### 查询流程

1. 用户右键托盘图标。
2. 点击 `快速查询端口`。
3. 弹出极简面板。
4. 用户输入端口号。
5. 点击 `查询`。
6. 后台线程调用 `inspect_ports([port])`。
7. 面板渲染结果列表。
8. 用户选择记录。
9. 点击 `终止进程`。
10. 调用 `kill_record(record)`。
11. 刷新查询结果。

### 权限不足流程

1. 用户以普通用户身份运行。
2. 查询到可疑占用，但终止失败或受限。
3. 面板显示：`当前不是管理员，无法终止该进程`。
4. 提供 `以管理员身份运行` 按钮。
5. 调用 Windows UAC 提权启动新实例。

## 极简面板设计

建议窗口尺寸：

```text
宽度：420 px
高度：360 px
```

布局：

```text
PortClear 快速清理
[ 端口号输入框          ] [查询]

结果
PID      进程名        状态        类型
1234     node.exe      LISTENING   user_process

[终止选中进程] [HTTP.sys 诊断] [打开主窗口]

日志/提示
当前权限：普通用户
```

设计原则：

- 不做复杂配置。
- 不展示大量技术解释。
- 不默认执行危险操作。
- `PID 4`、`TIME_WAIT`、无 PID 记录不能直接终止。
- 终止进程前建议弹出一次确认。

## 托盘菜单设计

推荐菜单：

```text
PortClear
当前权限：管理员/普通用户
----------------------
快速查询端口
打开主窗口
以管理员身份运行
----------------------
退出
```

行为：

- `快速查询端口`：打开小面板。
- `打开主窗口`：打开现有完整 GUI。
- `以管理员身份运行`：触发 UAC 重新启动。
- `退出`：关闭托盘图标和所有窗口。

## 管理员启动方案

Windows 下可以使用：

```python
ctypes.windll.shell32.ShellExecuteW(
    None,
    "runas",
    sys.executable,
    " ".join(sys.argv),
    None,
    1,
)
```

注意：

- `runas` 会触发 UAC。
- 用户取消 UAC 时不能视为程序错误。
- 如果是 PyInstaller 打包后的 EXE，应使用 `sys.executable` 直接重启当前 EXE。
- 如果是开发模式，应保证参数能正确传递。

## 打包影响

PyInstaller 需要把托盘图标和依赖一起打包：

```bash
python -m pip install pyinstaller pystray pillow
python -m PyInstaller -F -w main.py -n PortClear --add-data "assets/portclear.ico;assets"
```

如果 `pyinstaller` 命令无法识别，不要直接依赖 PATH，使用 `python -m PyInstaller` 更稳。这样可以确保 PyInstaller 安装在当前正在使用的 Python/Conda 环境里。

如果后续新增 `--tray` 默认启动模式，建议提供两个打包产物：

```text
PortClear.exe            # 默认托盘常驻
PortClear-GUI.exe        # 默认打开完整窗口，可选
```

也可以保留一个 EXE，通过启动参数区分：

```bash
PortClear.exe --tray
PortClear.exe --gui
```

## 风险点

### 托盘事件循环与 Tk 主循环冲突

`pystray` 有自己的事件循环，`tkinter` 也有主循环。建议：

- 主线程运行 Tk。
- 托盘图标运行在后台线程。
- 托盘菜单回调通过 `root.after()` 切回 Tk 线程打开面板。

或者：

- 主线程运行 `pystray.Icon.run()`。
- 每次需要 UI 时创建独立 Tk 窗口。

更推荐第一种，因为当前项目已经有 Tk GUI。

### 查询操作阻塞 UI

不能在 UI 线程直接执行端口查询。当前主窗口已经改为后台线程查询，托盘小面板也应该保持一致：

- 点击查询后禁用按钮。
- 后台线程调用 `inspect_ports()`。
- 查询完成后用 `root.after()` 更新 UI。

### 多窗口和重复实例

托盘常驻后，用户可能重复点击打开多个快速面板。建议：

- 快速面板只允许一个实例。
- 如果已经打开，则把现有窗口置顶。
- 主窗口也尽量只允许一个实例。

### 权限误解

非管理员模式下，用户可能以为工具“杀不掉进程”是 Bug。建议界面明确显示：

```text
当前权限：普通用户。只能处理当前用户有权限终止的进程。
```

### PID 4 误操作

托盘极简面板不能为了简化而省略安全限制。必须保留：

- 禁止终止 PID 4。
- 显示 HTTP.sys 诊断入口。
- 提示使用管理员权限定位具体服务。

## 推荐实现步骤

1. 新增 `windows/quick_panel.py`，先实现不带托盘的小面板。
2. 复用 `inspect_ports()` 和 `kill_record()`，完成端口查询与终止。
3. 新增 `windows/tray.py`，接入 `pystray` 托盘图标和菜单。
4. 修改 `main.py`，支持 `--tray` 和 `--gui`。
5. 添加 `assets/portclear.ico`。
6. 更新 `README.md` 的 Windows 使用和打包说明。
7. 做打包验证，确认托盘图标、UAC 提权、退出逻辑正常。

## 实施优先级

建议按以下优先级推进：

| 优先级 | 功能 | 原因 |
| --- | --- | --- |
| P0 | 快速查询面板 | 先验证核心交互是否舒服 |
| P0 | 后台线程查询 | 避免托盘面板未响应 |
| P0 | 终止选中进程 | 满足核心目标 |
| P1 | 托盘图标和右键菜单 | 完成常驻体验 |
| P1 | 以管理员身份运行 | 解决权限不足场景 |
| P2 | 打开完整主窗口 | 连接高级功能 |
| P2 | HTTP.sys 诊断弹窗 | 支持 PID 4 场景 |
| P3 | 开机启动 | 可选，不建议首版默认开启 |

## 最终建议

建议做。

这个优化和 PortClear 的定位非常匹配：主窗口负责完整诊断，托盘面板负责高频、快速、轻量操作。实现成本不高，核心逻辑可以大量复用，主要新增的是 Windows 托盘适配和极简 UI。

推荐首版不要追求复杂的“任务栏扩展面板”视觉效果，而是先做稳定的 Windows 托盘图标加小型弹窗。这样最容易打包、最容易维护，也最符合端口清理工具的实际使用习惯。
